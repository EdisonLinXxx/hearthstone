# 炉石传说自动对战 Bot 任务清单

## 1. 当前范围

- 运行环境固定：Windows 前台截图 + 前台输入
- 当前支持两个固定分辨率 profile：`1600x900`、`1440x900`
- 游戏模式为窗口化，窗口目标位置为 `(20, 20)`
- 目标模式为休闲模式
- 通过全局热键 `F8` 停止

## 2. 当前运行方式

主程序：
- `python -m bot.main --deck-index 1 --resolution 1600x900`
- `python -m bot.main --deck-index 1 --resolution 1440x900`
- `python -m bot.main --deck-index 1 --resolution 1440x900 --ocr-auto-sample`

标注工具：
- `python -m bot.ocr_label_mana_main`
- `python -m bot.ocr_label_cost_main`

anomaly 索引 / 过滤：
- `python -m bot.build_anomaly_manifest --resolution 1440x900 --tag ocr_anomaly`
- `python -m bot.build_anomaly_manifest --resolution 1440x900 --tag ocr_anomaly --stats`
- `python -m bot.build_anomaly_manifest --resolution 1440x900 --tag ocr_anomaly --has-trigger-reason ocr_untrusted`
- `python -m bot.build_anomaly_manifest --resolution 1440x900 --tag ocr_anomaly --only-zero-final-with-debug`

## 3. 优先级 1-5 落地状态

### [x] 优先级 1：统一手牌识别主链路
已落地：
- `parse_board_state()` 只负责 battle 基础状态
- battle 决策最终使用的 `hand_cards` 仅来自 OCR / cost gem 链路

- runtime / rules / board_state 职责边界已收口
- 日志已区分 final decision state 与 debug-only legacy candidates

### [x] 优先级 2：mana / cost 分离 OCR 配置
已落地：
- `mana` / `cost` 使用独立 OCR 配置
- `recognize_mana()` / `recognize_cost()` 分别走各自配置路径
- `bot/ocr/1440x900.yaml`、`bot/ocr/1600x900.yaml` 已更新

### [x] 优先级 3：费用宝石检测改为“候选 + 验证”两阶段
已落地第一版：
- gem 宽松生成候选
- 再做验证与过滤
- 已加入蓝色占比、白字占比、结构稳定性、综合分数等约束

### [x] 优先级 4：OCR 拒识与规则校验机制
已落地第一版：
- mana / cost OCR 支持显式拒识
- 已加入 best diff 过大拒识
- 已加入 best / second-best 差距过小拒识
- 已加入 mana / cost 合理范围校验
- 已加入轻量帧间异常跳变校验
- OCR 不可信时策略层不会继续出牌

### [x] 优先级 5：样本标注闭环（已推进到 P5-3）

#### [x] P5-1：异常优先采样

#### [x] P5-2：元信息规范化与 anomaly 索引
- `meta.json` 字段更统一
- 支持 `all_trigger_reasons`

#### [x] P5-3：anomaly 过滤 / 汇总 CLI
- `build_anomaly_manifest` 支持按 trigger / source / scene / trusted 等条件过滤

## 6. 当前交接重点

### Battle 决策
- 最终 `hand_cards` 只来自 OCR / cost gem 链路
- 不再依赖绿色高亮作为正式 playable 判定
- legacy 绿色候选仅用于 debug 观察

### OCR 可信性
- OCR 结果不是“永远强制输出”
- 当前已支持拒识、规则校验、轻量时序校验
- OCR 不可信时，runtime / rules 会等待而不是错误出牌

### 样本闭环
- anomaly 样本有独立目录
- anomaly 样本带 `meta.json`
- 可构建 `anomaly_index.csv`
- 可直接用 CLI 按 trigger / source / cards / reject reason 过滤和汇总
