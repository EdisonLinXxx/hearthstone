# VMware 部署与验收 Checklist

## 1. 宿主机准备

- [ ] 已安装 `VMware Workstation`
- [ ] 宿主机内存不少于 `16GB`
- [ ] 已为虚拟机预留至少 `80GB` 磁盘空间
- [ ] 已为虚拟机规划至少 `4` 核 CPU
- [ ] 已为虚拟机规划至少 `8GB` 内存

## 2. 虚拟机系统准备

- [ ] 已安装 `Windows 10` 或 `Windows 11`
- [ ] 已安装 `VMware Tools`
- [ ] Windows 显示缩放已设置为 `100%`
- [ ] 已关闭会导致分辨率自动变化的设置
- [ ] 虚拟机桌面分辨率已固定为 `1600x900`

## 3. 虚拟机软件安装

- [ ] 已安装 `Battle.net`
- [ ] 已安装 `Hearthstone`
- [ ] 已安装 `Python 3.10`
- [ ] 已将项目拷贝到虚拟机内
- [ ] 已执行依赖安装：

```powershell
pip install -r requirements.txt
```

## 4. 游戏环境确认

- [ ] 炉石客户端语言为简体中文
- [ ] 炉石分辨率为 `1600x900`
- [ ] 炉石显示模式为窗口化
- [ ] 炉石窗口未最大化
- [ ] 炉石可以从 Battle.net 正常启动

## 5. 项目启动前检查

- [ ] 工程目录存在
- [ ] `requirements.txt` 安装完成
- [ ] 可在虚拟机内打开 PowerShell
- [ ] 可进入项目目录

## 6. 主程序启动验证

- [ ] 已执行命令：

```powershell
python -m bot.main --deck-index 1
```

- [ ] 启动日志中能看到窗口识别成功
- [ ] 启动日志中能看到窗口尺寸为 `1600x900`
- [ ] 炉石窗口会自动归位到 `(20, 20)`
- [ ] `F8` 可以停止脚本

## 7. 采样工具验证

- [ ] 已执行采样命令：

```powershell
python -m bot.sample_main --tag in_battle --deck-index 1
```

- [ ] `bot/samples/1600x900` 下有新样本生成
- [ ] 完整窗口截图正常
- [ ] 区域裁剪图正常

## 8. 队列流程验证

- [ ] 可以从主界面点击进入对战
- [ ] 可以点击 `传统对战`
- [ ] 可以进入选套牌页面
- [ ] `deck_index=1-9` 能选中对应套牌
- [ ] 可以点击 `开始`
- [ ] 能成功进入匹配

## 9. 对局流程验证

- [ ] 起手换牌页面能自动点击确认
- [ ] 能正常进入正式对局
- [ ] 自己回合时会尝试出牌
- [ ] 无牌可出时会点击结束回合
- [ ] 对手回合时不会误点结束回合

## 10. 结算流程验证

- [ ] 胜利结算页可自动点击继续
- [ ] 失败结算页可自动点击继续
- [ ] 后续确认页可自动点击确认
- [ ] 能回到选套牌页面
- [ ] 能再次开始下一局

## 11. 宿主机共存验证

- [ ] Bot 运行在虚拟机内时，不影响宿主机外部鼠标使用
- [ ] 宿主机外可正常浏览网页
- [ ] 宿主机外可正常看直播
- [ ] 不需要频繁切回虚拟机干预

## 12. 高风险检查项

- [ ] VMware 未自动改动虚拟机分辨率
- [ ] VMware 未自动改动客户机缩放
- [ ] 炉石窗口位置稳定
- [ ] 点击坐标未明显偏移
- [ ] 结果页不会长时间卡住
- [ ] 鼠标停靠点逻辑正常，未出现 hover 干扰

## 13. 若出现问题，优先排查

- [ ] 分辨率是否仍为 `1600x900`
- [ ] Windows 缩放是否仍为 `100%`
- [ ] 炉石是否仍为窗口化
- [ ] VMware 是否开启了自动缩放
- [ ] 是否需要在虚拟机内重新采样模板
- [ ] 是否需要重新校准：
  - [ ] `traditional_battle_button`
  - [ ] `queue_play_button`
  - [ ] `end_turn`
  - [ ] `result_continue_button`

## 14. 最终验收

- [ ] 可在虚拟机内连续运行至少 `2` 局
- [ ] 宿主机外部操作基本不受影响
- [ ] 能通过 `F8` 安全停止
- [ ] 不需要人工干预即可完成一轮完整闭环
